Introduction¶

The NY Metropolitan Transport Authority provides detailed data files on transit usage to the public to encourage developers to build application and tools to help MTA riders. For this project we are looking at a dataset that combines MTA subway turnstile usage with weather reports from wunderground.com to determine if more people ride the subway when it is raining versus when it is not raining.

Statistical Test¶

The statistical hypothesis we will test is shown below.

$H_0$ : the population mean for rider traffic is the same for both rain and dry conditions

$H_a$ : the population mean for rider traffic is greater for rain conditions vs dry conditions.

Before we get to that we need to examine the quality of data and possibly make some transformations to remove any obvious inconsistencies that might skew our test result.

For the discussion in this IPython Notebook we'll import in a number of Python libraries. Most were used throughout the Intro to Data Science course, but I've added a few that add some convenience functions for regression and visualization.

patsy
- creates a model design from a text formula describing the relationship between explanatory and response variables. Statsmodels has variants of regression models that allow for use of pasty formulas.
seaborn
- is a visualization library built on matplotlib. It provides improvements to existing plotting functions and adds a number of plot types that we'll use below.

Linear Regression¶

Now let's see if we can define a reasonable predictive model for rider entries using the attributes in the improved dataset.

To do that we'll use patsy to define design matrices using R-like formulas. By intuition, volume should depend on time of day, day of week, weather conditions and location. That model is tested below using statsmodels ordinary least squares (OLS) regression.

One advantage of the pasty formula is it automatically creates dummy variables from categorical data & removes one of the values to avoid multicollinearity (where predictor variables are correlated).

In the results summary of this model, the coefficient of determination, $R^2$ is 0.548. 54.8% of the variance of ENTRIESn_hourly can be explained by the attributes we chose in the model.

All predictors in this model are categorical and weather conditions turns out to not be that significant in the model. If 'conds' is removed from the formula, $R^2$ drops to 0.546. Intuitively this makes sense - the factors associated people going to and from work (day of week & time of day) overwhelms the weather factor.

Conclusion¶

In this study we examined two datasets of MTA turnstile traffic. In the first dataset, two groups of entry traffic were separated into rain and norain days. At that level of granularity, the null hypothesis that the mean daily entries were the same could not be rejected.

The improved dataset provided more accurate weather conditions and allowed for comparison of entry values that were collected over the same interval. That dataset showed that higher entry values can be correlated with rainy weather.

The linear regression model showed that the dominant predictors for traffic was time of day and day of week. Weather was not nearly as significant in the model.

Reflection¶

When examining the original data set, I originally rolled up all entry totals to daily values, since the rain attribute was the same for all units and time of day. This resulted in a RAIN sample size of 10 and an NO RAIN sample size of 20. Both distributions were not normal and were relatively small - I didn't feel I could use either the Mann-Whitney or T-Test due to the size. To get a larger sample number I used Daily Unit entries instead. This is an area for future learning - what kind of transformations can be done on data sets and how do they impact tests and conclusions?

Mann-Whitney U test did not work properly on my windows machine. I originally tried to use it to compare distributions since it is more efficient for non-normal datasets, but the function returned 'nan' for the p value. Another area for learning is selecting the best test for various sample sizes.

For the improved dataset, my solution for getting comparable entry values was to omit data that did not rollup entry counts over 4 hours. I thought about converting values to true hourly rate, but I still would not be able to reconstruct the missing 'conds' value - so I felt it was better to just omit the data to remove any skew that it could cause.

Visualization is another area for future exploration as well. I used seaborn's boxplot and violin plots to compare the rain and norain groups. Since the data is highly positively skewed the upper whisker was quite long and it was difficult to compare the quartiles. I could have tried to use a log of the values to make the comparison easier. Seaborn also has many plots for models that use categorical data. I really just scratched the surface in this exercise.
